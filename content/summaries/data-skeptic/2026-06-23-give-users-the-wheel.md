# 把方向盘交给用户

- Source: https://dataskeptic.com/blog/episodes/2026/give-users-the-wheel

## 一句话结论
这期的核心观点很直接：推荐系统不该只靠点赞、停留和历史行为“猜”用户意图，而应该让用户能用自然语言直接下指令；Fuyuan Lyu 提出的 DPR（Decoupled Promptable Sequential Recommendation）是在不推翻现有推荐引擎的前提下，把 LLM 作为“用户意图接口”接到传统推荐模型上。

## 这期在讲什么
Kyle Polich 和 Fuyuan Lyu 讨论的是一个很现实的问题：今天的推荐系统很强，但用户几乎没有方向盘。你想让 YouTube 白天多推技术新闻、晚上少推同类内容，或者明确说“别再给我推这个了”，现有系统往往只能靠隐式行为慢慢猜。

Fuyuan 的研究 Paper《Give Users the Wheel》提出 DPR 框架：把用户的自然语言提示词和顺序推荐系统解耦处理，再融合到一个统一的推荐决策里。它不是用 LLM 取代协同过滤，而是让 LLM 负责理解“show me more of this / don’t recommend that”这类指令，让传统推荐模型继续发挥排序和召回的优势。

## 核心要点

### 1. 用户真正想要的不是“更聪明的猜测”，而是可表达的控制权
节目里一个很重要的动机是：用户的偏好是上下文相关的，不是静态标签。比如同一个人早上想看科技新闻，晚上想看猫视频，单靠点赞/不点赞很难表达这种切换。

这意味着推荐系统的目标不只是“预测下一条内容”，还要支持用户即时修正系统对自己的理解。自然语言在这里的价值，不是更花哨的交互，而是把用户从被动反馈者变成主动操作者。

### 2. DPR 的关键是“解耦”，不是推倒重来
DPR（Decoupled Promptable Sequential Recommendation）的思路是把两个能力拆开：一边是语言模型，负责理解 prompt；另一边是传统 sequential recommendation，负责基于历史行为建模并做推荐。

这种设计很务实。现有推荐系统已经在召回、排序、在线服务上积累了大量工程和数据优势，完全用 LLM 替换并不现实。DPR 试图保留这些资产，只是在决策入口加上自然语言控制层，让系统能响应“更像这个”“不要那个”这样的增量指令。

### 3. 最大难点不是模型结构，而是如何评估“用户指令被正确执行”
对话里提到一个现实问题：如果没有现成数据集，怎么评估一个“可被 prompt 驱动”的推荐系统？因为大多数推荐数据只记录点击、观看、评分，并不记录用户曾经发过什么自然语言指令。

这类研究会遇到典型的冷启动评估困境：你想验证系统能不能按指令调节推荐，但训练和测试数据里根本缺少这种监督信号。所以论文和实验设计必须想办法构造任务、定义目标，或者在有限数据上近似模拟用户意图。

### 4. LLM 在这里更像“意图解析器”，不是最终推荐器
这期一个值得注意的边界是：LLM 的作用主要是理解用户说的话，把模糊的自然语言转成系统可执行的控制信号，而不是直接生成推荐列表。

这种分工很重要。因为推荐系统的强项在于利用大规模行为数据做个性化排序；LLM 的强项在于语言理解和交互。把两者拼接，能避免“用生成式模型硬做推荐”带来的不可控性，也更符合现有平台的演进路径。

### 5. 这类交互可能改变 YouTube、TikTok 和新闻信息流的产品形态
如果用户真的能用对话 steering feed，产品逻辑会从“系统猜你喜欢什么”转向“系统理解你现在想要什么”。这对短视频、新闻流、播客和内容平台都很关键，因为这些场景里用户意图变化快，且对信息污染和兴趣漂移更敏感。

但这也意味着产品要重新设计交互：用户不是只点 thumbs-up / thumbs-down，而是可能说“今天别给我娱乐内容”“我现在只想看入门教程”。这会把推荐系统从黑箱行为预测，推向一种半显式、可协商的内容分发机制。

## 对 AI 从业者的启发
1. 推荐系统的下一阶段，可能不是“更准”，而是“更可控”。  
2. LLM 最适合切入的地方，往往不是替代核心排序，而是补上自然语言接口层。  
3. 做产品时要区分“模型能力”与“用户控制能力”，后者常常被低估。  
4. 评估新型交互系统时，数据缺失本身就是研究问题，不只是工程问题。  
5. 对内容平台来说，给用户方向盘，可能比继续增强算法更能提升长期信任。

## 值得继续追问
- DPR 在真实线上系统里会如何和召回、排序、重排模块协同？
- 用户的自然语言指令如何被安全地约束，避免被恶意 prompt 干扰推荐？
- 这种“可对话的推荐”能否在低频场景、长内容场景里比短视频更快落地？
- 如果用户频繁改写偏好，系统如何判断这是短期意图还是长期兴趣变化？
- 评价“用户获得更多控制权”时，除了点击率，还应该看哪些长期指标？
